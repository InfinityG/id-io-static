About

What is id-io?

  • A simple yet powerful Federated Identity Provider
  • Provides a non-proprietary 'Single Sign On' ('SSO') service
  • Securely and privately stores client registries and personal meta-data
  • A core authentication mechanism for all your digitally connected applications
  • The primary record is encrypted and stored on a cryptographically secure, distributed ledger ("blockchain")

Why use it?

  • Open Source, so you can see the code. Fork it from Github for your own use. Extend and improve
  • Simple to install and use
  • Built with Ruby and Sinatra, using modern coding standards. Deployable with Docker onto Linux platforms
  • Cryptographically secure - uses 256 bit encryption and signing algorithms
  • Built to securely manage private core identities on the blockchain

Features

A RESTful API which supports

  • Creation of trusts between id-io API and Relying Parties
  • Registration of user identities
  • Authentication of users
  • Generation of signed and encrypted tokens for consumption by Relying Parties

Basic SSO Functionality

  • Supports username/password AND username/signature combinations
  • Generates encrypted and signed tokens with limited lifespan
  • Token data includes role information

Strong Encryption

  • Tokens are signed with an ECDSA private key
  • Tokens are encrypted with a shared 256 bit AES key specific to relying party

Relying Party Support

  • Tokens generated by the API can be parsed by relying parties using 'ig-identity-rp-validator' Ruby gem

Identity Storage

  • Local: MongoDB is used as the backing store for identities
  • Distributed: A blockchain is used for public identity storage (still in development)

Webhooks

  • Registrations on the API support a Webhook callback mechanism on successful registration
